# ComfyUI 工作流命名规范

## 目录规则

工作流必须放在对应的目录下，前端按目录筛选：

| 目录 | 用途 | 行为 |
|------|------|------|
| `文生图/` | 纯文本生图（无输入图片） | 不传图片，只注入提示词 |
| `图生图/` | 需要输入图片处理 | 传图片 + 注入提示词 |
| `根目录`（其他） | 不会出现在前端列表 | 仅在 ComfyUI 内手动打开使用 |

> 额外的 `文生图/` 和 `图生图/` 目录在 `WF_DIR_TXT2IMG` / `WF_DIR_IMG2IMG` 环境变量配置。

---

## 关键词跳过规则

文件名包含以下关键词的，**自动跳过提示词注入**，仅传图片：

```
洗图 | 高清 | clean | upscale | 放大 | 重绘 | 转真人
```

### 匹配这些关键词的工作流

| 文件名（含路径） | 关键词 | 系统行为 |
|----------------|--------|---------|
| `图生图/seedVR2高清放大.json` | `高清` `放大` | 跳过注入，只传图片 |
| `图生图/洗图工作流.json` | `洗图` | 跳过注入，只传图片 |
| `图生图/真人高清重绘.json` | `重绘` `高清` | 跳过注入，只传图片 |
| `图生图/次元重绘 w170-xxx.json` | `重绘` | 跳过注入，只传图片 |
| `图生图/二次元重绘放大xxx.json` | `重绘` `放大` | 跳过注入，只传图片 |
| `07.转真人 自动遮罩.json` | `转真人` | 跳过注入，只传图片 |

### 不匹配这些关键词的工作流

| 文件名 | 系统行为 |
|--------|---------|
| `文生图/anima双采样工作流.json` | 正常注入提示词 |
| `文生图/Qwen-三图.json` | 正常注入提示词 |
| `图生图/万物提取.json` | **报错**（无 CLIPTextEncode，需加关键词或修改） |

## 命名原则

### ① 需要提示词注入的（传统工作流）

```json
文生图/
├── anima双采样工作流.json       ← 正常注入 ✅
├── Qwen-三图.json              ← 正常注入 ✅
└── Z-image生图.json            ← 正常注入 ✅
```

命名随意，只要不含关键词即可。

### ② 不需要提示词注入的（纯处理/重绘/放大类）

```json
图生图/
├── seedVR2高清放大.json         ← 匹配"高清"+"放大" → 跳过 ✅
├── 次元重绘 w170-AniToon.json   ← 匹配"重绘" → 跳过 ✅
└── 真人高清重绘.json             ← 匹配"高清"+"重绘" → 跳过 ✅
```

**文件名必须包含至少一个关键词**，否则会报错。

### ③ 仅在 ComfyUI 内手动打开的（不出现在前端）

放在根目录（不在 `文生图/` 或 `图生图/` 下）：

```json
根目录/
├── 对比.json           ← 本地看图用，不出现在前端
```

## 新增工作流检查清单

```
□ 这个工作流需要提示词注入吗？
├─ 需要 → 放在文生图/ 或 图生图/ 目录，文件名任意
├─ 不需要 → 文件名包含关键词：洗图/高清/clean/upscale/放大/重绘/转真人
│
□ 需要用户上传图片吗？
├─ 需要 → 放 图生图/ 目录
├─ 不需要 → 放 文生图/ 目录
│
□ 只是本地手动用？
└─ 放根目录（不出现在前端）
```
